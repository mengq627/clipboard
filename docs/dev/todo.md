## 0.1.1
> 优化体验，修复bug

### optimize
- [ ] 界面的边框修改得稍微圆润一点
- [ ] 当内容列表刷新时，仅位置发生变化的内容刷新，而不是整个内容列表都刷新。（当前的实现方式是整个列表刷新，导致闪烁感明显，并且性能很差）

### feature
- [ ] 快捷键win +v 偶尔失灵：点击x关闭后，应用仍然被关闭。增加一个“当点击x时，最小化而不是退出”的开关，再开发此功能。 
- [ ] 增加一个开关：当回车选择内容后，是否自动粘贴
- [ ] 快捷键：列表支持vim风格的快捷键：<C-f>向下翻页，<C-b>向上翻页，gg到顶部，G到底部
- [ ] 支持搜索功能：在内容列表上方增加一个搜索框，支持文本模糊搜索
- [ ] 快捷键：支持P置顶，支持D删除

## 0.1.0
> 得让自己满意，达到自己可以使用的地步
> 修复极度影响用户使用体验的问题
> 打包，发布给用户使用
> 优先修复bug，从现在开始不使用AI直接开发
> 着眼于“剪切板”核心功能的开发。
> 保证可用性，完全可以使用，不影响核心功能使用体验

### optimize
- [x] 按钮悬停时，背景框圆润一点
- [x] 分组列表刷新：当内容分组时，仅分组的内容数量刷新，而不是整个分组的列表刷新
- [x] 当应用开启，但是失去焦点时，win+v重新聚焦，而不是隐藏应用
- [x] 监听系统剪切板从定时轮询修改为事件监听回调

### feature
- [x] UI 简化：所有的按钮默认不显示文字，而是以小图标显示，当鼠标悬停0.x秒，显示当前按钮的文字信息
- [x] 托盘逻辑优化：不显示在任务栏，只显示在托盘
- [x] 快捷键绑定：使用 win + v 展开：
- [x] 支持图片
- [x] 增加设置功能：设置快捷键，设置分组列表最大内容数等等
- [x] 主界面唤出后，支持Up/Down/j/k选择内容，回车确认选择，选择后自动隐藏界面
- [x] 支持excel等富文本/从系统剪切板获取文本内容

### bug

- [x] 点击最小化按钮时，并没有隐藏界面，而是出现在页面的左下角
- [x] 置顶按钮不生效，点击后，内容并不会置顶
- [x] 点击复制按钮后，分组列表会刷新，预期应用UI无变化
- [x] 点击托盘的显示界面后，界面很小，并且在左下角显示，预期应该和上一次隐藏界面时的大小位置一样。
- [x] 点击退出按钮，应用不退出
    看起来是点击x，应用就进入d状态了，快捷键就没有反应了。这要怎么断点呢？
    应用托盘和主界面是两个东西，托盘退出了，主界面也不会退出的
- [x] 应用会开启多个实例，预期只能有一个存在
- [x] 复制图片/文本时，存在报错
- [x] 应用打开后，失去焦点，再次使用快捷键唤出应用时，应用没有反应
- [x] 应用打开后，默认不处于可选择状态
